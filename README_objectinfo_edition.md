### 自定义 Champsim 对象追踪扩展

本仓库基于原仓库 ChampSim 进行了功能扩展，实现了对象级别的内存访问追踪和缓存统计，为内存访问行为分析提供更精细的粒度支持。
新增功能
1. 对象地址追踪
在生成 trace 时，跟踪 `malloc()` 函数的返回值以获取对象地址，并同步记录对象大小，为后续对象级追踪奠定基础。
2. CPU 流水线对象表
在 CPU 执行阶段，针对 `malloc` 指令生成对象表项（映射关系：object_addr -> ObjectInfo），用于全程追踪对象的生命周期及内存访问情况。
3. 缓存命中/缺失统计
对象在各级缓存（L1、L2、LLC）的访问命中和缺失情况会实时更新至对象表，实现对象粒度的缓存行为精准统计。
使用方法
1. 将测试源码放入指定目录：`mytests/test.cpp`
2. 执行运行脚本：
        ` 
        ./run.sh
        `
3. 程序执行完成后，终端会直接输出对象统计信息，包括：对象地址、对象大小、各级缓存命中次数、各级缓存缺失次数等核心数据。
实现细节
1. Tracer 插桩
文件路径：`tracer/pin/champsim_tracer.cpp`
- 对原始代码的 malloc() 和 free() 函数进行插桩处理，捕获对象的创建与释放事件。
- 新增 `VOID ImageLoad()` 函数，将每个 malloc() 的 size（对象大小）和 addr（对象地址）写入一条 trace 指令，该指令在 CPU 流水线阶段先于对应 malloc() 执行。
- 生成的 trace 数据将作为对象表构建的核心输入。
2. CPU 流水线对象管理
文件路径：`src/ooo_cpu.cc`
- 对象表定义：`std::unordered_map<uint64_t, ObjectInfo> history_table;`，其中 key 为对象虚拟地址，value 为对象的访存信息和缓存统计数据。
- 新增核心函数：
        
  - `handle_malloc_event()`：在 malloc 执行时，为新创建的对象初始化并添加对象表项。
  - `handle_free_event()`：在 free 执行时，删除对应对象的表项，释放资源并终止该对象的追踪。
- 在 CPU 流水线各个关键阶段，实时更新对象表，记录对象的完整生命周期信息（创建、访问、释放）。
3. 缓存访问统计
文件路径：`src/cache.cc`
- 修改核心函数 `try_hit()` 和 `handle_miss()`，增加对象表匹配逻辑：当访问地址存在于对象表中时，同步更新该对象表项的缓存命中/缺失计数。
- 支持对对象在各层缓存（L1、L2、LLC）的访问情况全面统计，包括：命中次数、缺失次数、访问总次数。
4. 统计输出
文件路径：`src/plain_printer.cc`
- 新增函数 `print_object_stats()`，用于打印对象表中所有有效对象的统计信息，输出内容包括：
        
  - 基础信息：对象大小、生命周期
  - 缓存统计：各级缓存的访问命中次数、缺失次数
